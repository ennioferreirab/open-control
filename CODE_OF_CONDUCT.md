# Code of Conduct

## Our Standard

We want Open Control to be a respectful, technically rigorous, and welcoming
project for contributors of all experience levels.

Examples of expected behavior:

- being respectful in technical disagreement
- assuming good intent while giving direct feedback
- sharing context, evidence, and reproduction steps
- helping others learn the codebase and workflow

Examples of unacceptable behavior:

- harassment, discrimination, or intimidation
- personal attacks or demeaning language
- publishing private information without permission
- derailing discussions in ways that block collaboration

## Scope

This code of conduct applies to repository discussions, issues, pull requests,
and other project-managed community spaces.

## Reporting

If you experience or witness unacceptable behavior, contact the project
maintainers privately. Public escalation should be avoided when it would expose
private information or make the situation worse.

## Enforcement

Maintainers may remove comments, close threads, or restrict participation when
behavior violates this code of conduct.
