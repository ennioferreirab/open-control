# Security Policy

## Supported Scope

Security reports are welcome for code maintained in this repository, including:

- `mc/`
- `dashboard/`
- `shared/`
- repository-owned scripts and docs that affect runtime behavior

Reports for `vendor/nanobot/` may still be useful, but upstream fixes might need
to land in the source project before they can be adopted here.

## Reporting A Vulnerability

Please do not open a public GitHub issue for a suspected vulnerability.

Instead:

1. Contact the repository maintainers privately through the project's security
   reporting channel or direct maintainer contact.
2. Include a clear description of the issue, impact, affected files or flows,
   and reproduction steps.
3. Share any suggested mitigation if you already have one.

If a dedicated security mailbox or GitHub Security Advisory workflow is added
later, this document should be updated to point to it directly.

## Response Expectations

Maintainers should:

- acknowledge receipt within a reasonable timeframe
- validate severity and scope
- coordinate a fix and disclosure plan
- publish release notes when the fix ships
